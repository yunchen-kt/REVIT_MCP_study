| 已合併 PR | #14 — Domain 新增（詳圖同步、自動標註、視圖裁切、樓梯可視化） |

### 核心問題

Jacky 使用 Revit 2020，專案主線支援 2022-2026。他的 C# 程式碼有 Revit 2020 專用的修改（`Floor.Create` 條件編譯、`IdType` 相容性、`CreateDetailLines` partial class 調整）。

**直接 `git pull` 會覆蓋這些修改，導致編譯失敗。**

### 解決方案

設定 Git 的 local merge driver，讓 `git pull` 時自動保護 C# 程式碼，只更新知識層。

### Step 1：設定 upstream remote


---

### Step 2：設定 Safe Pull 保護

這一步設定 Git 的 local merge driver，讓以下 C# 檔案在 `git pull` 時自動保留 Jacky 的版本。

**2-A：建立 keep-mine merge driver**

```bash
git config merge.keep-mine.name "Always keep local version"
git config merge.keep-mine.driver "true"
```

> 說明：`"true"` 是 Unix 指令，永遠回傳成功。效果是合併時不做任何事，保留本地版本。

**2-B：建立 .git/info/attributes**

```bash
mkdir -p .git/info

cat > .git/info/attributes << 'ATTRS'
# === Revit 2020 Safe Pull Protection ===
# 這些檔案在 git pull 時會自動保留 Jacky 的版本
# 此設定只存在本機，不會推到 GitHub

# C# 程式碼（有 Revit 2020 特定修改）
MCP/Core/CommandExecutor.cs merge=keep-mine
MCP/Core/Commands/CommandExecutor.CurtainWall.cs merge=keep-mine
MCP/Core/Commands/CommandExecutor.SmokeExhaust.cs merge=keep-mine
MCP/Core/RevitCompatibility.cs merge=keep-mine
MCP/Application.cs merge=keep-mine

# Build 設定（沒有 Release.R20）
MCP/RevitMCP.csproj merge=keep-mine
MCP/RevitMCP.addin merge=keep-mine
ATTRS
```

**2-C：驗證設定**

```bash
# 確認 merge driver 已設定
git config merge.keep-mine.driver
# 應該輸出: true

# 確認 attributes 檔案存在且內容正確
cat .git/info/attributes
# 應該看到 7 個檔案標記為 merge=keep-mine
```

### Step 4：同步後驗證

**4-A：檢查 CLAUDE.md 的數字是否需要調整**

```bash
# Jacky 實際有幾個工具？
grep -c "case \"" MCP/Core/CommandExecutor.cs
# 如果跟 CLAUDE.md 裡的 58 不同，需要修正 CLAUDE.md 裡的數字

# Jacky 實際有幾個 Skill？
ls .claude/skills/*/SKILL.md 2>/dev/null | wc -l

# Jacky 實際有幾個 Domain？
ls domain/*.md 2>/dev/null | wc -l
```

如果數字不一致，修正 CLAUDE.md 裡對應的數字（搜尋 `58 個工具`、`17 個`、`24 個`）。

**4-B：確認新的 Skill 是否引用了 Jacky 沒有的工具**

```bash
# 列出所有 Skill 引用的工具名稱
grep -h "工具：\|tools:" .claude/skills/*/SKILL.md | head -20

# 對照 Jacky 的 CommandExecutor 有哪些 case
grep "case \"" MCP/Core/CommandExecutor.cs | sed 's/.*case "//;s/".*//' | sort
```

如果某個 Skill 引用的工具在 Jacky 的 CommandExecutor 裡不存在，**不需要刪除 Skill**，只是那個 Skill 在 Revit 2020 環境無法完整執行。

**4-C：確認 Hook 是否正常運作**

```bash
# 測試 hook 偵測腳本
echo '{"tool_name":"Bash","tool_input":{"command":"git merge test"},"tool_response":{}}' | \
  .claude/hooks/detect-claudemd-trigger.sh

# 應該看到 JSON 輸出，包含「合併外部 PR」字樣
```

## 提 PR 時的注意事項

### ✅ 歡迎提交

- `domain/*.md` — 新的法規 SOP
- `.claude/skills/*/SKILL.md` — 新的 Skill 定義
- `domain/lessons.md` — 開發經驗

### ⚠️ 需要討論

- C# 新增的 command — 建議只提交**新增的方法**，不要提交整個 CommandExecutor.cs
- 工具定義 `.ts` — 可以提，但要確認 upstream 的 C# 端有對應實作

### ❌ 不要提交

- `RevitMCP.csproj` 的版本修改
- `bin/`、`obj/`、`scratch/` 等暫存目錄
- 個人設定檔（`.env`、`settings.local.json`）

### PR 提交前檢查

```bash
# 確認沒有夾帶不該提交的檔案
git diff --cached --name-only
git diff --cached --name-only | grep -E '\.(dll|exe|pdb)' && echo "⚠ 有二進位檔案！"
```
