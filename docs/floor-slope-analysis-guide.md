# analyze_floor_slopes 使用手冊 — 樓板排水坡度分析

> 對應工具：`analyze_floor_slopes` ｜ SOP：[`domain/floor-slope-analysis.md`](../domain/floor-slope-analysis.md) ｜ 原始需求：Issue #45（yunchen-kt）

室外樓板（陽台、露台、屋頂層、車道）要確認排水坡度有沒有做到設計標準（常見最小 1～2%）。過去這件事要一片一片樓板點開、量法線、心算坡度。這個工具讓你用一句話，把選到的樓板（或全部室外樓板）的坡度一次算完，並把結果寫回參數。

---

## 1. 它到底在做什麼

對每一片樓板：

1. 取出樓板的實體幾何（`Solid`）。
2. 找出所有「朝上的平面」（頂面）。
3. 用頂面法向量與垂直方向的夾角，換算成坡度百分比。
4. 回報這片樓板的**最小／最大坡度**，並（可選）寫回你指定的參數。

換句話說，你早就知道怎麼判斷一片樓板有沒有洩水——這個工具只是把「逐片量、逐片算」這串你會做但很煩的事，變成一句話就跑完。

---

## 2. 怎麼叫它

它是 MCP 工具，所以你是用自然語言請 AI 呼叫，不需要記參數名。例如：

- 「幫我分析目前選取的這幾片樓板的排水坡度。」
  → AI 帶上你選取元素的 ID 呼叫工具。
- 「把這個模型所有室外樓板的坡度算一遍，結果寫到 Comments。」
  → AI 不帶 elementIds，工具自動收集所有 `Function = Exterior` 的樓板。
- 「分析這片露台的坡度，把數值寫到『洩水坡度』這個參數。」
  → AI 帶上 `paramName = "洩水坡度"`。

---

## 3. 參數

| 參數 | 必填 | 說明 |
|---|---|---|
| `elementIds` | 否 | 要分析的樓板 Element ID 陣列。**省略**時，工具自動收集所有型別 `Function = Exterior` 的樓板。 |
| `paramName` | 否 | 坡度回寫的目標參數名稱。**預設 `Comments`**。 |

---

## 4. 看懂回傳結果

工具回傳像這樣的結構：

```jsonc
{
  "Success": true,
  "ProcessedCount": 3,
  "ParameterName": "Comments",
  "Floors": [
    {
      "ElementId": 123456,
      "MinSlopePercent": 1.50,
      "MaxSlopePercent": 2.30,
      "UpwardFaceCount": 2,
      "Written": "Slope 1.50%~2.30%"
    }
  ],
  "Errors": []
}
```

- **`MinSlopePercent` / `MaxSlopePercent`**：這片樓板所有朝上頂面的坡度範圍。一片做了洩水的樓板，最小與最大會落在你設計的坡度區間；一片完全水平的樓板，兩者都會接近 `0`。
- **`UpwardFaceCount`**：被納入計算的朝上面數量。
- **`Written`**：實際寫回參數的字串；若該片無法回寫，會列在 `Errors`。
- **`Success`**：只有當所有樓板都成功（含回寫）才是 `true`；有任何一片進 `Errors` 就是 `false`，但其它片仍照常處理與回傳。

### 回寫的格式

- 目標是**文字參數**（如 `Comments`）：寫入 `Slope {min}%~{max}%`。
- 目標是**數值參數**：寫入最大坡度的**比例值**（例如 2% → `0.02`）。
- 目標**不存在／唯讀／型別不符**：不寫，該片原因列在 `Errors`，數值仍照常在 `Floors` 回傳。

## 5. 搭配上色，把不合規的樓板標出來

拿到每片的坡度後，常見的下一步是「把坡度低於設計最小值的樓板染色」。這時接 `override_element_graphics`，對應 [`domain/element-coloring-workflow.md`](../domain/element-coloring-workflow.md)。例如：

> 「剛剛分析完，把最大坡度小於 1% 的樓板塗紅，讓我一眼看到哪幾片洩水不夠。」

---

## 6. 限制與注意事項

- **只認平面頂面**：算法找的是 `PlanarFace`。彎曲頂面的樓板無法取得單一法向量，會被列入 `Errors`（不中斷整批）。曲面坡度要另以 UV 取樣評估，本工具未涵蓋。
- **朝上面門檻**：只納入法向量與水平夾角 45° 以內（`n.Z > 0.7`）的頂面。這是為了排除板側、倒角這類近垂直的面（它們會吐出數百 % 的假坡度）。排水屬緩坡，這個門檻不會誤殺真正的洩水面；但若你要分析的是 45° 以上的陡坡，它會被排除。
- **數值參數的單位**：寫入數值參數時用的是「比例值」，工具不會替你做 Revit 內部單位換算。若你的目標參數是角度或長度型，顯示值的語意可能不如預期——**最穩的做法是用文字參數（如 `Comments`）**，或自建一個無單位的 Number 共享參數。
- **室外判定看型別**：自動收集是讀樓板**型別**的 `Function` 參數。如果專案沒把型別 Function 設成 Exterior，請改用顯式選取（讓 AI 帶 `elementIds`）。

---

## 7. 背後的算法（想深入的人看）

對每片朝上頂面，取正規化法向量 `n`：

- `n.Z` 即法向量與垂直 Z 軸夾角的餘弦。
- 坡度角 `θ = acos(n.Z)`。
- 坡度百分比 `= tan(θ) × 100`。

水平面 `n.Z = 1 → θ = 0 → 坡度 0%`。幾何走訪會遞迴進入 `GeometryInstance`，避免被 instance 包住的頂面被漏算。完整步驟見 [`domain/floor-slope-analysis.md`](../domain/floor-slope-analysis.md)。
