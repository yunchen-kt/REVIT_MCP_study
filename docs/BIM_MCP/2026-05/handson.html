5/22 DRY-RUN LESSONS

5 條 dry-run 揭露的新 lessons

5/22 跑完整 6 步 demo dry-run 揭露 5 個系統性發現。其中 L-025 升格為第四憲法。

L-025 · Active State Re-Anchoring（升憲法級）

使用者刻意在 dry-run 中切換多個視圖（1F→6F→2F）不告知 AI。AI 用 session 開頭的 viewId 做新 claim → 跟使用者眼前畫面脫鉤。升格為第四憲法：每個 stateful claim 前必須重 anchor，雙向 context sync 協議。

→ 第四憲法詳細

L-026 · Tool Scope Mismatch

Step 3 五工具批次中，check_exterior_wall_openings 是 project-wide（不吃 level 參數），其他 4 個是 level-scoped。AI 若統一呈現為「2FL 報告」會誤導——整案違規含跨樓層開口。

→ domain/tool-capability-boundary.md L7

L-027 · Regulation Type → Coloring Strategy

染色策略不能跨規範類型通用：wall-anchored（§45/§110）直接染 violation 牆段；room-anchored（§41 採光、排煙）沒有「違規牆段」，需 hosting walls proxy。

→ domain/tool-capability-boundary.md L8

L-028 · MCP Failure Mode & Recovery

5/22 dry-run 中段連續 2 次 get_active_view timeout。AI 拒絕用 session memory 推測視圖（避免 stale snapshot 染色）→ 等使用者點任意視圖重建 focus 修復。

→ domain/tool-capability-boundary.md L9

L-029 · BIM 模型內在不一致

日本 BIM 模板的 Room 同時有「面積」（Revit 計算）和「面積 部屋 調整値」（手填校正）兩個值，差 0.5-5%。在 2% 排煙 / 12.5% 採光的邊界 case 上可能跨越合規門檻。AI 不該替使用者選一個值——必須兩個都端出來。

→ domain/daylight-area-check.md 末段 + domain/smoke-exhaust-review.md 末段
