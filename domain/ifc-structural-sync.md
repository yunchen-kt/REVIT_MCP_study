# IFC 鋼構同步與幾何校正 SOP

## 1. 目標

將 IFC 或 linked model 中的鋼構樑轉換為 Revit 原生結構構件時，避免常見的幾何偏移與高度錯誤。此 SOP 著重於同步前的幾何判讀、類型建立、放置座標與批次更新策略。

## 2. 適用情境

- 連結模型中的鋼構樑需要在主模型中轉為可編輯的 Revit 原生構件。
- IFC 匯入後的元素缺乏穩定 `LocationCurve`。
- 直接抓最長邊或 BoundingBox 中線會導致 XY 偏移。
- 樑頂需要對齊主模型樓板底面。
- 同一批樑需要可重跑、可更新、可追蹤來源。

## 3. 核心原則

### 3.1 不信任單一 Location 屬性

IFC 或 linked element 常見問題是 `LocationCurve` 不存在或不代表真實中心線。不要只依賴單一 `Location` 屬性，也不要直接取最長邊當作樑中心線。

建議順序：

1. 優先檢查是否有可靠 `LocationCurve`。
2. 若無，抽取幾何邊線與 solids。
3. 以樑長向邊線分組，估算截面重心。
4. 以重心連線作為原生樑放置路徑。

### 3.2 以截面重心修正 XY 偏移

IFC H 型鋼常會讓最長邊落在翼板外緣。若用外緣線放置原生樑，會產生固定方向的 XY 偏移。較穩定的做法是：

- 先判斷樑主方向。
- 收集與主方向平行的長邊。
- 計算長邊中點的平均座標。
- 用平均座標代表截面重心。
- 以兩端截面重心建立中心線。

### 3.3 樑頂對齊樓板底面

若要讓樑頂貼齊樓板底面，不應從樑重心直接向上射線，因為樑可能已經穿入樓板，射線會先打到樓板頂面或內部面。

建議流程：

1. 從樑中心線中點取得 XY 座標。
2. 將射線起點放在樑重心下方一段安全距離，例如 3m。
3. 對主模型樓板向上射線。
4. 第一個命中的樓板下表面作為目標高度。
5. 依樑型深度回推中心線 Z 值。

### 3.4 預先校正再建立構件

大批量建立構件時，先算好正確座標再建立，比建立後再移動穩定。這能降低 Revit join、group、constraint 與 regenerate 帶來的錯誤。

## 4. 類型建立策略

### 4.1 尺寸來源

類型尺寸可來自 IFC 參數、族群名稱、DXF 鋼構表或人工對照表。不要固定假設表格圖層名稱，例如 `S-TABLE`；應先列出候選圖層並讓使用者確認。

### 4.2 命名規則

建議命名包含來源範圍、構件編號與尺寸：

```text
[ALL/Floor]-編號-尺寸 cm
```

範例：

```text
ALL-B12-H400x200x8x13
F3-B07-H300x150x6.5x9
```

### 4.3 重複類型處理

若多樓層使用相同尺寸，優先共用同一類型。若同一編號在不同樓層代表不同尺寸，需在類型名稱加入樓層或來源標記，避免覆蓋。

## 5. 批次同步策略

### 5.1 可重跑標記

建立原生構件時，應在可追蹤欄位寫入來源資訊，例如：

```text
Link:<LINK_ID>|Kind:BEAM|Source:<SOURCE_ID>
```

重跑前先掃描相同 `Link` 與 `Kind` 的舊構件，讓同步具有冪等性。

### 5.2 分批交易

大量構件不宜全部包在單一 transaction 中。建議每 50 到 100 根樑作為一批，批次完成後再 regenerate。若某批失敗，較容易定位問題來源。

### 5.3 失敗處理

同步時常見警告包含元素 join、群組修改、重疊幾何或約束失敗。應記錄錯誤 ElementId 與來源 ID，並避免因單一構件失敗中止整批流程。

## 6. 驗證清單

- 原生樑中心線與 IFC 樑視覺重心一致。
- 樑頂高度與樓板底面一致，沒有貼到樓板頂面。
- 重跑同步不會產生重複構件。
- 類型尺寸與 IFC、DXF 或人工對照表一致。
- 批次同步後能追溯每根樑的 linked source ID。
- 錯誤構件有記錄，方便人工複核。

## 7. 常見問題

### 7.1 樑整排偏移

通常是中心線取到翼板邊緣，而不是截面重心。應改用多條長向邊線的中點平均值，或直接從 solid 幾何估算中心。

### 7.2 樑高度貼到樓板頂面

通常是射線起點太高或位於樓板內。將射線起點下移，再向上尋找第一個樓板面，可降低誤判。

### 7.3 同步後產生重複樑

通常是缺少來源標記或重跑前沒有清理舊構件。應使用穩定的 `Link + Kind + Source` 標記作為可重跑識別。
