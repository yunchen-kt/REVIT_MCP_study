# 元素查詢參數工作流程 (Element Query Workflow)

本文件定義在 Revit 模型中正確查詢元素屬性的標準作業流程，涵蓋實例與類型參數的區分、常見混淆場景的澄清，適用所有 Revit 版本（2022~2026）。

---

## 核心觀念：實例參數 vs 類型參數

Revit 元素的屬性資料分為兩個**截然不同的查詢層級**：

| 分類 | 英文名 | 說明 | 範例 |
|------|------|------|------|
| **實例 (Instance)** | Instance Parameter | 每個元素個別擁有的值 | 標記 (Mark)、位置座標、高程偏移 |
| **類型 (Type)** | Type Parameter | 同類型所有元素共享的值 | 類型標記 (Type Mark)、尺寸規格、Fire Rating |

> [!WARNING]
> **常見誤解**：使用者在標籤 (Tag) 或明細表中看到的「編號」，可能是 Mark（實例參數）也可能是 Type Mark（類型參數）。
> 混淆 Mark 與 Type Mark 是最常見的**零結果查詢**原因。

---

## 查詢工作流程

```
步驟 1                    步驟 2                    步驟 3
+---------------+        +---------------+        +---------------------+
| 取得模型概覽  | -----> | 列出欄位清單  | -----> | 判斷 Instance       |
|               |        |               |        | vs Type 參數        |
| get_active    |        | get_category  |        |                     |
|   _schema     |        |   _fields     |        | (根據欄位/AI 判斷)  |
+---------------+        +---------------+        +----------+----------+
                                                             |
步驟 5                    步驟 4                             |
+---------------+        +---------------------+            |
| 檢視單一元素  | <----- | 送出條件式查詢       | <----------+
|               |        | 取得符合條件的元素   |
| get_element   |        |                     |
|   _info       |        | query_elements      |
+---------------+        |   _with_filter      |
                         +---------------------+
```

### 步驟詳述

| 步驟 | 工具 | 目的 | 備註 |
|------|------|------|------|
| 1 | `get_active_schema` | 取得目前視圖或模型中各類別的元素數量概覽 | 同時可確認查詢範圍是否包含目標類別 |
| 2 | `get_category_fields` | 列出特定類別所有可用欄位的完整清單 | 確認欄位名稱是否正確（注意英文名 vs 中文名） |
| 3 | 判斷 | 根據欄位清單及使用者需求判斷應查實例或類型 | 無法確定時兩者都嘗試 |
| 4 | `query_elements_with_filter` | 根據條件式查詢取得符合的元素清單 | 支援 equals / contains / greater / less 等運算 |
| 5 | `get_element_info` | 查看單一元素的所有詳細屬性 | 用於取得完整欄位值供深入分析 |

## 常見混淆欄位對照表

| 使用者描述 | 正確欄位名（通常） | 分類 | 判斷依據 |
|---------|-----------|------|---------|
| 某型號的所有元素在哪 | **類型標記 (Type Mark)** | Type | 同型號的元素共享此值 |
| 某個特定元素的編號 | **標記 (Mark)** | Instance | 每個元素各自的識別碼 |
| 房間的編號或代碼 | **號碼 (Number)** | Instance | 房間專用欄位 |
| 房間的名字 | **名稱 (Name)** | Instance | 房間專用欄位 |
| 標籤上顯示的值 | **類型標記 / 標記** | 視情況 | 需確認 Tag 設定的參數後才能確定 |

---

## 查詢結果為零時的排查路徑

當查詢回傳 `Count: 0` 或找不到任何匹配元素時：

```
標記 (Mark) 找不到？--> 嘗試類型標記 (Type Mark) 找不到？--> 嘗試備註 (Comments)
                                                                       |
                                                                  仍找不到？--> 確認是否為
                                                                                連結模型元素
                                                                                (參照 tool-capability-boundary.md)
                                                                                或樓層/視圖範圍問題
```

## 常見類別及其關鍵參數

本表列出最常查詢的類別及其重要參數，方便快速參考。

| 類別 | InternalName | 常用查詢參數 |
|------|-------------|------------|
| 窗戶 | Windows | 類型標記, 標記, 窗台高度, 框架寬度, 框架高度 |
| 門 | Doors | 類型標記, 標記, 框架寬度, 框架高度, Fire Rating |
| 牆壁 | Walls | 類型標記, 框架寬度, Fire Rating, 長度 |
| 柱子 | Columns | 類型標記, 標記, 底部樓層 |
| 樓梯 | Stairs | 階梯實際深度, 階梯實際高度, 階梯實際踏面深度 |
| 房間 | Rooms | 名稱, 號碼, 面積 |

> [!NOTE]
> **跨版本注意**：所有 Revit 版本（2022~2026）的查詢流程相同。
> 唯一差異為 Revit 2025+ 使用 `ElementId=long`（透過 `RevitCompatibility.cs` 處理），不影響查詢工作流程。
